About The Structure


Status: Finished
Tools used: Unity, C#, Visual Studio and Git
Roles: Game Programmer, Game Designer and Project Manager

While developing The Structure with a team of 5, I implemented multiple systems in the game based on the designs that the team and I made. Throughout the project, I helped the team in numerous ways like making extensive user stories, controlling project scope, helping teammates solve problems related to coding and solving merge conflicts.

My Contributions:

  • Boss finite state machine including animations
  • Damage falloff and numbers
  • Rockslide attack using a particle system
  • Player SphereCast
  • Improved the camera
  • Collisions with objects
  • Gameplay design

Boss finite state machine

I wrote a finite state machine for the boss which includes the various states that the boss has. While writing the state machine in C# I made sure to write it so more states could be added later on. This state machine controls the several animations and transitions in the animator state machine and provides one access point for the boss state to be controlled.

State Machine code on GitHub



Damage fallof and numbers

I developed a system that changes the damage that the player does based on the distance between the player and the enemy. When a bullet hits an enemy a number is shown displaying the damage that the bullet did, based on the damage the color also changes.

Damage Fallof code on GitHub



Rockslide attack

I implemented a Rockslide attack in the game by making use of the particle system in Unity. I changed these particles to fit the theme of the Rockslide and also added shadows and collisions to it.

Particle Collision code on GitHub



Player SphereCast

The player in this game makes use of a kinematic Rigidbody which means that collisions won't work as intended. To prevent the player from leaving the arena I cast a sphere in front of the player that detects objects with the environment layer mask and exited out of the move method if the sphere collides with a GameObject.

SphereCast code on GitHub



Improved camera

I wrote a script that zooms the camera out based on the distance between the player and the boss. I also made the camera follow the player smoothly known as camera lerp.

Camera code on GitHub



Player and Boss Collision

I wrote scripts that handle the collisions for both the player and the boss. In these scripts, I also added conditions and subtracted health if an attack landed.
